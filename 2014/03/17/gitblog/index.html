GitBlog的原理大概就类似于一个静态空间，所以你的blog也当然就是静态的了。加上GitHub的质量似乎很棒。
所以说白了，GitBlog就是通过本地的静态bolg生成html然后再git到github.com的自定义页面去，就成了博客。

本屌用的静态博客生成工具是hexo，用node.js写的一个东西，似乎是评价最好的。本屌就以此为例。

不过忽然想到大牛用的linux，各种东西怎么安装大牛自己想办法吧。
需要这俩：

  • node.js
  • git

0x01
首先，github的配置：

创建一个仓库（就是github的一个项目），命名为「用户名.github.io」。

建立SSH key，在git的安装目录执行如下命令：

设置邮箱与用户名：

git config --global user.email "nixi@dasb.me"
git config --global user.name "nixiteam"

生成密钥：

ssh-keygen -t rsa -C "nixi@dasb.me"

会提示输入密码，本屌记得留空就行。

大牛自己找生成的ssh.pub在哪。找到之后复制内容，添加到Add SSH Key。

完成之后使用：

ssh -T git@github.com

github配置完成，验证不成功了再说，本屌先往下说。

0x02
其次，hexo的配置

npm install -g hexo  	//安装hexo
hexo init  				//部署hexo到当前文件夹

这样其实就完成了。
有些具体的设置还要在_config.yml文件中改。

比如：
安装一个漂亮的主题Light：

git clone git://github.com/tommy351/hexo-theme-light.git themes/light

在配置文件中修改相应设置。
还有类似评论，RSS，和github绑定等，我在贴出来的配置文件中说明。

把我的_config.yml贴上，让大牛参照参照：

0x05
hexo的常用命令：

hexo n "文章名"			
//新建文章，之后打开「source/_posts/文章名.md」添加内容。支持HTML与markdown语法。

hexo server
//本地运行预览。

hexo d -g
//生成静态文件并发布到github。就不用git再提交了！
